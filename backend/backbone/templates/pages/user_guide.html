1. Quick Start

Install dependencies and run your app:

# Install
uv sync
# or: pip install -r requirements.txt

# Copy environment file
cp .env.example .env
# → Edit .env and set SECRET_KEY, MONGODB_URL, ADMIN_EMAIL, ADMIN_PASSWORD

# Run
uv run uvicorn main:app --reload

Then visit:

  • /docs — Interactive OpenAPI documentation
  • /admin — Admin panel (use ADMIN_EMAIL / ADMIN_PASSWORD)
  • /pages/user-guide — This page

2. Defining Models

Backbone provides two base document classes:

  • AuditDocument — adds created_at, updated_at, soft-delete (is_deleted), and signals.
  • BackboneDocument — extends AuditDocument with automatic slug generation.
from backbone.domain.base import BackboneDocument
from backbone.domain.models import User
from beanie import Link, Indexed
from pydantic import Field
from typing import Optional, List

class Article(BackboneDocument):
    title: Indexed(str) = Field(max_length=200)
    body: str
    status: str = Field(default="draft")
    author: Optional[Link[User]] = None

    # Automatic slug from title
    slug: str = Field(
        default="",
        json_schema_extra={"slugify": True, "populate_from": "title"},
    )

    class Settings:
        name = "articles"   # MongoDB collection name

AuditDocument fields available automatically: created_at, updated_at, created_by, updated_by, is_deleted, deleted_at, deleted_by

3. Generic CRUD Views

Inherit GenericCrudView to auto-generate 5 routes:

GET / — Paginated list
POST / — Create
GET /{id} — Retrieve
PATCH /{id} — Partial update
DELETE /{id} — Soft delete
from backbone import GenericCrudView, IsAuthenticated

class ArticleAPI(GenericCrudView):
    model = Article
    search_fields = ["title", "body"]          # ?search=...
    filter_fields = ["status", "author"]        # ?status=published
    lookup_field = "slug"                       # GET /api/articles/my-slug
    fetch_links = True                          # Auto-resolve Link fields
    permission_classes = [IsAuthenticated]

# Mount the router
app.include_router(ArticleAPI.as_router(
    prefix="/api/articles",
    tags=["Articles"]
))

Extending Lifecycle Hooks

class ArticleAPI(GenericCrudView):
    model = Article

    async def before_create(self, data, user):
        # Inject the logged-in user as author
        data["author"] = str(user.id)
        return data

    async def after_create(self, instance, user):
        # Fire a background task after creation
        from backbone.services.tasks import task_service
        task_service.run_in_background(notify_subscribers, article_id=str(instance.id))
        return instance

    async def get_queryset(self, request, user):
        # Scope list to only published articles for non-admins
        if user and user.is_admin():
            return {"is_deleted": {"$ne": True}}
        return {"is_deleted": {"$ne": True}, "status": "published"}

4. Filtering & Search

Backbone supports rich URL-based filtering when fields are listed in filter_fields:

URL param MongoDB equivalent
?status=published{"status": "published"}
?price__gt=100{"price": {"$gt": 100}}
?price__gte=50&price__lte=200{"price": {"$gte": 50, "$lte": 200}}
?status__in=draft,published{"status": {"$in": ["draft","published"]}}
?title__contains=guide{"title": {"$regex": "guide", "$options": "i"}}
?title__startswith=How{"title": {"$regex": "^How", "$options": "i"}}
?search=backbone$or across all search_fields
?sort=-created_atsort by created_at descending
?page=2&page_size=20skip=20, limit=20

5. Permissions

ClassDescription
AllowAnyNo restriction (default)
IsAuthenticatedValid JWT required (401 otherwise)
IsStaffUserrole in {STAFF, ADMIN, SUPERUSER}
IsAdminUserrole in {ADMIN, SUPERUSER}
IsSuperUserrole == SUPERUSER
IsOwnercreated_by == current user id
 
from backbone import IsAuthenticated, IsAdminUser, IsOwner

# Combine with & (AND) or | (OR)
class MyView(GenericCrudView):
    permission_classes = [IsAuthenticated & IsOwner]

class AdminView(GenericCrudView):
    permission_classes = [IsAdminUser | IsOwner]

# Custom permission
from backbone import BasePermission
class IsPublished(BasePermission):
    async def has_object_permission(self, obj):
        return getattr(obj, "status", None) == "published"

7. Email Verification

Enable email verification in your .env:

REQUIRE_EMAIL_VERIFICATION=true
EMAIL_ENABLED=true
EMAIL_HOST=smtp.gmail.com
EMAIL_USERNAME=yourapp@gmail.com
EMAIL_PASSWORD=yourapppassword

When enabled:

  1. User registers → receives verification email
  2. User clicks link → /pages/verify-email?token=...
  3. User is now verified and can log in
  4. Login attempts by unverified users receive HTTP 403

Template overrides & custom layouts

Backbone ships default Jinja2 templates inside the installed package. Your application can override any of them without changing Python code: create the same relative path under your project's working directory (where you run uvicorn), and that file wins.

Area Your project path (first) Packaged fallback
Admin UI templates/admin/<name>.html backbone/templates/admin/
Public pages templates/pages/… backbone/templates/pages/…
Transactional email templates/email/<name>.html backbone/templates/email/

How it works: routers use a Jinja2 ChoiceLoader with <cwd>/templates/… first, then the package directory from BackboneSettings.backbone_templates_path. Only files you add under your project need to exist; everything else is served from Backbone.

Custom-only templates: you can add new templates under templates/pages/ for your own routes and render them with the same Jinja2Templates pattern (app loader first). Extend pages/base_public.html for a consistent public layout.

Context variables: admin views pass model metadata, forms, and admin_site globals; public auth pages expect keys documented next to each route in backbone/web/routers/pages.py; email bodies use the arguments of MailService.send_* methods.

8. Admin Panel

Register any model to manage it via the web admin at /admin:

from backbone import admin_site, ModelAdmin

class ArticleAdmin(ModelAdmin):
    list_display = ["title", "status", "created_at"]
    search_fields = ["title"]
    ordering = "-created_at"

admin_site.register(Article, admin_class=ArticleAdmin, category="Content")

# Then call setup_backbone before starting the app:
setup_backbone(app, models=[Article])

Features: list, search, sort, create, update, file upload (auto Attachment), linked-field dropdowns, delete.

9. Signals

React to model lifecycle events anywhere in your codebase:

from backbone import signals
from myapp.models import Order

@signals.post_create.connect(Order)
async def on_order_created(instance, **kwargs):
    await send_order_confirmation_email(instance)

@signals.on_field_change.connect(Order)
async def on_order_status_changed(instance, changed_fields, **kwargs):
    if "status" in changed_fields:
        old_status, new_status = changed_fields["status"]
        if new_status == "shipped":
            await send_shipping_notification(instance)

@signals.post_delete.connect(Order)
async def on_order_deleted(instance, **kwargs):
    await cleanup_order_resources(instance)

10. Background Tasks

Two modes available:

from backbone.services.tasks import task_service

# Mode 1: In-process (asyncio) — fire and forget, no Redis needed
task_service.run_in_background(send_welcome_email, to_email=user.email)

# Mode 2: Redis queue — durable, survives restarts
await task_service.enqueue("myapp.jobs.generate_report", user_id=str(user.id))

# Start the worker (separate terminal or process):
# python -m backbone.cli worker

Worker tasks must be referenced by their full Python dotted path (e.g. myapp.jobs.my_function). All task runs are logged to the task_logs MongoDB collection and visible in the admin.

11. Cron Scheduler

Schedule recurring jobs using decorator syntax. The scheduler starts automatically with setup_backbone().

from backbone.services.scheduler import scheduler

# Run at midnight every day
@scheduler.cron(hour=0, minute=0)
async def daily_report():
    await generate_and_email_daily_report()

# Run every 15 minutes
@scheduler.interval(minutes=15)
async def refresh_cache():
    await rebuild_homepage_cache()

# Run every Sunday at 3am
@scheduler.cron(day_of_week="sun", hour=3, minute=0)
async def weekly_cleanup():
    await purge_soft_deleted_records()

# Programmatic (no decorator)
scheduler.add_cron_job(my_function, hour=2, minute=30)
scheduler.add_interval_job(another_function, seconds=30)

All times are UTC. Requires apscheduler>=3.10 (already in dependencies).

12. CLI Tools

# Change any user's password

# List all users
python -m backbone.cli list-users

python -m backbone.cli worker --concurrency 4

python -m backbone.cli run-tests

python -m backbone.cli info
